About Ape Framework

Ape is the Python-native smart contract framework that lets you compile, test, deploy, and analyze contracts across multiple chains.

Our Mission

Our mission is to make Web3 development feel natural for Python developers. Ape is designed to give protocol engineers, researchers, and data scientists a powerful but approachable toolkit for exploring, building, and shipping on-chain systems.

Built for Python Developers

Ape is for anyone who thinks in Python and wants to work with smart contracts and on-chain data.

Python developers exploring Web3 for the first time

Data scientists and researchers analyzing on-chain activity in Jupyter notebooks

Smart contract engineers who prefer Python workflows and pytest-style testing

Teams migrating from older Python tooling like Brownie

Why Ape Exists

Ape was created to fill the gap for a complete, Python-first environment for building, testing, and deploying smart contracts across multiple chains. Instead of juggling separate scripts, CLIs, and SDKs, Ape brings accounts, networks, plugins, and contract lifecycle management together in one cohesive workflow.

What Makes Ape Different

Ape combines a full smart contract development environment with the power of the Python ecosystem.

Python-native workflow

Develop and test contracts using Python, pytest, and familiar tooling.

Unified multi-chain development

Work with local forks, testnets, and mainnets in a single command-line session.

Modular plugin ecosystem

Extend Ape with plugins for contract languages, explorers, and wallet integrations.

End-to-end contract lifecycle

Compile, test, deploy, verify, and manage contracts from one framework.

Production-grade debugging

Gain insight into transactions with integrated tracing, gas reporting, and decoded calldata.

Powered by the Python data stack

Use Ape alongside Jupyter, Pandas, NumPy, and other scientific libraries.
