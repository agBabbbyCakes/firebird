About Ape Framework

Ape is the Python-native smart contract framework that lets you compile, test, deploy, and analyze contracts across multiple chains — using the tools you already know and love from the Python ecosystem.

Ape Framework gives Python developers a unified, extensible environment for building Web3 applications. From local forks and testnets to live production networks, Ape offers a consistent workflow for scripting, testing, deployment, and debugging. With a modular plugin system and deep integration with Python’s data science stack, Ape bridges the gap between on-chain engineering and off-chain analysis.

Our Mission

Our mission is to make Web3 development feel natural for Python developers. Ape is designed to give protocol engineers, researchers, and data scientists a powerful but approachable toolkit for exploring, building, and shipping on-chain systems — without forcing them to switch languages or abandon their existing workflows.

Built for Python Developers

Ape is for anyone who thinks in Python and wants to work with smart contracts and on-chain data.

Python developers exploring Web3 for the first time

Data scientists and researchers analyzing on-chain activity in Jupyter notebooks

Smart contract engineers who prefer Python workflows and pytest-style testing

Teams migrating from older Python tooling like Brownie

Infrastructure and backend teams building bots, services, and automation in Python

Why Ape Exists

The Web3 ecosystem has world-class tooling for JavaScript and Rust — frameworks like Hardhat and Foundry — but Python developers have historically been under-served. Earlier tools like Brownie showed what a Python-native experience could look like, but many teams now need a modern, actively maintained framework that keeps pace with new networks and standards.

Ape was created to fill this gap: a complete, Python-first environment for building, testing, and deploying smart contracts across multiple chains. Instead of juggling separate scripts, CLIs, and SDKs, Ape brings accounts, networks, plugins, and contract lifecycle management together in one cohesive workflow.

If you’re coming from Brownie or other Python tools, Ape provides familiar patterns with a more modern plugin system, broader chain support, and active development.

What Makes Ape Different

Ape combines a full smart contract development environment with the power of the Python ecosystem.

Python-native workflow

Develop and test contracts using Python, pytest, and familiar tooling — no need to switch to a new language or stack.

Unified multi-chain development

Work with local forks, testnets, and mainnets in a single command-line session, with straightforward configuration and scripting.

Modular plugin ecosystem

Extend Ape with plugins for contract languages, explorers, wallet integrations, and multiple chains, including both EVM and non-EVM networks.

End-to-end contract lifecycle

Compile, test, deploy, verify, and manage contracts from one framework, including rich scripting and console support.

Production-grade debugging

Gain insight into transactions with integrated tracing, gas reporting, and decoded calldata to debug both locally and on live networks.

Powered by the Python data stack

Use Ape alongside Jupyter, Pandas, NumPy, and other scientific libraries to analyze on-chain data, prototype strategies, and build research workflows.

Open Source and Community-Driven

We welcome contributions, feedback, and new plugins that expand what’s possible with Python on-chain.
